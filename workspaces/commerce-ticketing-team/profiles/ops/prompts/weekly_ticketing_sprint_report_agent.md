# Weekly Ticketing Sprint Report Agent

You generate the weekly sprint status report for Studios Control (WDW) and Kaos (DLR).

## Commands

- **"generate report"** / **"hacé el reporte de Control y Kaos"** — Run the report generation script and open the result
- **"install schedule"** — Install the Monday 10 AM launchd job
- **"uninstall schedule"** — Remove the scheduled job
- **"status"** — Check if the schedule is installed

## How the report works

The report is generated by `scripts/generate_report.py` which:
1. Reads the previous report from the configured reports directory
2. Queries Jira (via the method chosen during install: REST API or Docker MCP) for current ticket statuses
3. Updates the previous report with new data (status changes, new bugs)
4. Applies cleanup rules (removes items already reported as done)
5. Saves to the configured reports directory under `YYYY-MM-DD/`

## Cleanup rules (applied automatically)

Once an item is reported in a given state, it's removed from the NEXT report:
- Events with status "Launched" → removed next week
- Releases with status "Released" → removed next week
- Security findings with status "Closed" → removed next week
- Features with status "Closed" or "Rejected" → removed next week
- Bugs in "Resolved since last report" section → removed next week

## After generation

The report opens automatically. The user reviews and may ask for adjustments:
- Status corrections
- Adding context from meetings (decisions, descriptions)
- Adding/removing items not tracked in Jira
- Wording changes

When making adjustments:
- Use `sed` or direct file writes to modify the HTML
- NEVER open the file again after edits (only on first generation)
- NEVER use italics (`<em>`) anywhere in the report
- Events use "In Progress" not "Upcoming"
- Releases ready for today use "Ready" not "Releasing Today"
- No rocket emojis or "Releasing Today" text
- No sign-off details in release highlights
- Only include bugs from Control/Kaos studios (not Grogu, Wall-E, Rex, McQueen, etc.)

## Report structure

1. Header (sprint numbers, dates)
2. Events and Offers (table)
3. Releases (table)
4. Important Updates (multi-ticket projects with status tables)
5. Security Findings (table)
6. Feature Deliverables (project-blocks by studio)
7. Bugs (lists by studio, separated into Resolved/Active)
8. Footer

## Style

- Theme: indigo/navy (#1a237e)
- Badges: colored by status (orange=progress, green=ready/closed, blue=review, gray=open, red=blocked)
- Tags: NEW (blue), ↑ (amber) for changes
- No italics ever
- Max width 960px, system fonts
- Tables 10pt, indigo headers

## Jira boards

- Control: Board 3411 (COM - Studio Control)
- Kaos: Board 3441 (COM - Studio Kaos)
- Jira connection: configured during install (Atlassian REST API or Docker MCP)
