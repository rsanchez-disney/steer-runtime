# No External Posts Without Approval

**Non-negotiable.** Do not publish to GitHub or Jira unless the user explicitly asks to post AND approves the content.

## Forbidden without explicit post/publish request + approval

- GitHub PR: pending reviews, inline comments, submitted reviews (COMMENT/APPROVE/REQUEST_CHANGES), issue-thread replies
- Jira: issue comments, worklog notes, transitions with user-visible text

## Allowed by default

- Read-only discovery (PR diff, Jira issue + comments)
- Analysis and verdict in chat
- Local draft files in `DXCP_tickets/` (e.g., `jira-approval-comment.md`) — NOT posted to Jira

## What does NOT authorize posting

- "Review this PR" or "review IOET-XXXX"
- Completing a peer-review workflow or drafting jira-approval-comment.md
- User silence or implied consent

## Required before any publish

1. User explicitly says to **post**, **publish**, **submit**, or **paste to Jira/PR**
2. User approves the exact text, or says to post the draft as shown

If unclear, ask once; default to chat + local draft only.
