# fnb-shared — Architecture Guide

## Overview

Shared library consumed by Mobile Order (MO), Dine Mobile Check-in (DMC), and Merchandise Mobile Checkout (MMC). Contains reusable infrastructure code shared across all FNB iOS apps.

## Module Structure

```
FNBShared/
├── Network/                        # Shared networking infrastructure
├── Logging/                        # Logging infrastructure
│   └── AccessControl/              # Log access control rules
├── Location/                       # Geolocation services
├── RemoteConfiguration/            # Remote document fetching
├── FeatureToggles/                 # Feature flag system
│   ├── Settings/                   # Toggle types (String, Integer, Toggle, etc.)
│   ├── Storage/                    # Local persistence
│   └── CommonUI/                   # Settings UI components
├── SwiftUI/                        # Shared SwiftUI components
├── Utils/                          # Shared utilities
├── Multiplatform/                  # KMP interop (experimental)
│   ├── common/
│   ├── mobileorder/
│   └── dinecheckin/
├── Accessibility/                  # Accessibility helpers
├── CreditCard.xcassets/            # Credit card icons
└── FNBShared.docc/                 # Documentation catalog

FNBSharedTests/                     # Unit tests
├── Mocks/                          # SwiftierMocky auto-generated mocks
└── (mirrors FNBShared structure)

Example/                            # Demo host app
```

## What Belongs Here

- Code used by **2+ FNB apps**
- Shared networking infrastructure (`NetworkWorker`, `NetworkService`)
- Location validation and fetching
- Logging infrastructure (events, breadcrumbs, remote logs)
- Feature toggle system
- Remote configuration fetching
- Shared SwiftUI components and styles
- Utility extensions and helpers

## What Does NOT Belong Here

- Feature-specific business logic
- UI that's only used in one app
- App-specific configuration
- Domain models specific to MO, DMC, or MMC

## Key Modules

### Network

Provides the base networking layer used by all FNB apps:

```swift
public protocol NetworkWorkerProtocol {
    func execute<T: Decodable>(_ request: NetworkRequest) async throws -> T
    func execute(_ request: NetworkRequest) async throws -> [AnyHashable: Any]
    func execute(_ request: NetworkRequest) async throws
}
```

- `NetworkWorker` — async/await based HTTP client
- `NetworkService` — URLSession wrapper
- `NetworkRequest` — request builder (URL, method, headers, body)
- `NetworkError` — typed error cases

### Feature Toggles

A full feature flag system with:
- Multiple value types: Toggle (bool), String, Integer, StringList, Picker, Location, Blocklist
- `LocalStorageProtocol` / `UserDefaultsLocalStorage` for persistence
- SwiftUI settings UI (`SettingView`, `TriStateToggleView`)

### Location

Reusable location services:
- `LocationValidator` — validates guest proximity
- `CurrentLocationFetcher` — one-shot location provider with factory pattern
- Override decorator for testing/debug

### Logging

Shared logging infrastructure:
- `EventLogger` — event tracking
- `RemoteLog` — New Relic remote logging model
- `AccessControl/` — rules-based log filtering

## Mocking (Tests)

Uses **SwiftierMocky** (via `Mockfile`). Auto-generated mocks output to `FNBSharedTests/Mocks/AutogeneratedMocks.swift`.

## Dependency Management

- **SPM packages** managed via Xcode (stored in `.xcodeproj`)
- **Sharon** (legacy, deprecated): `Sharon.toml` at repo root
- Dependencies: `WDPRFinderCore`, `WDPRUIKit`

## Versioning & Integration

- Consuming apps depend on `FNBShared` via Sharon (currently version `97.0.0`)
- SPM mirror: `studio-anglerfish/fnb-shared-spm`
- Default branch: `main`

## Pre-Build Steps

Before building the project, always run these steps on changed files:

1. **Regenerate mocks** — if any new or modified protocol has `AutoMockable` conformance, run `swiftiermocky generate` to update `FNBSharedTests/Mocks/AutogeneratedMocks.swift`.
2. **Format changed files** — run `xcrun swift-format format --in-place` on all modified files.

## Notable Conventions

- **Public API surface** — all types/methods intended for consuming apps must be `public`
- `NetworkWorkerProtocol` is the shared base for all network calls across apps
- `Clock` protocol abstracts time for testability
- Feature toggles use SwiftUI views in `FeatureToggles/CommonUI/`
- `Multiplatform/` contains KMP interop code (experimental, not widely used)
- Documentation catalog at `FNBShared.docc/`
