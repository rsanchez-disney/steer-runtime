# Cursor IDE Setup

Get the same coding standards, MCP integrations, and project context from steer-runtime in Cursor IDE.

## Prerequisites

- [Cursor IDE](https://cursor.sh/) installed
- steer-runtime cloned locally
- MCP servers installed (`./setup.sh mcp-install`) — optional but recommended

## Quick Start

```bash
cd ~/steer-runtime

# Install rules + MCP config to your project
./setup.sh cursor install ~/my-project

# Add project-specific context (optional)
./setup.sh cursor init-memory ~/my-project
```

This creates:
```
~/my-project/.cursor/
├── mcp.json                    # MCP server configs (Jira, Confluence, MyWiki, GitHub, Mermaid)
└── rules/
    ├── 00-golden-rules.mdc     # Always-on: coding standards
    ├── 01-project-mappings.mdc # Always-on: Jira → repo mappings
    ├── 02-conventional-commits.mdc # Always-on: commit format
    ├── 10-java-backend.mdc     # Activates on *.java, pom.xml
    ├── 11-node-webapi.mdc      # Activates on controllers, routes
    ├── 12-angular-ui.mdc       # Activates on *.component.ts
    ├── 13-go-services.mdc      # Activates on *.go
    ├── 14-flutter-mobile.mdc   # Activates on *.dart
    ├── 20-testing-standards.mdc # Activates on test/spec files
    ├── 21-security-guidelines.mdc # Always-on: security rules
    ├── 22-architecture-patterns.mdc # Manual: architecture guidance
    ├── 30-ba-guidelines.mdc    # Manual: BA/PO workflows
    ├── 31-qa-guidelines.mdc    # Manual: QA workflows
    ├── 32-ops-guidelines.mdc   # Manual: ops workflows
    ├── 33-pm-guidelines.mdc    # Manual: PM/scrum workflows
    ├── 40-guardrails.mdc       # Always-on: write protection
    ├── 50-sdlc-workflow.mdc    # Manual: implementation workflow
    ├── 51-pr-template.mdc      # Manual: PR format
    ├── 52-story-analysis.mdc   # Manual: Jira analysis
    └── 60-project-context.mdc  # Generated by init-memory (not installed by default)
```

## Commands

```bash
./setup.sh cursor install <dir>       # Install rules + MCP config
./setup.sh cursor sync <dir>          # Update rules from latest templates
./setup.sh cursor remove <dir>        # Remove .cursor/ directory
./setup.sh cursor init-memory <dir>   # Generate project context rule
```

## How Rules Work

Cursor rules use `.mdc` files with frontmatter:

- `alwaysApply: true` — rule is always active (golden rules, security, guardrails)
- `globs: ["**/*.java"]` — rule activates when editing matching files
- Neither set — rule is available for manual activation via `@` mention

## MCP Integration

The `cursor install` command generates `.cursor/mcp.json` pointing to the same MCP servers Kiro uses. If you've already run `./setup.sh mcp-install`, tokens are picked up automatically.

Available MCP tools in Cursor:
- **Jira** — fetch stories, create issues, manage sprints
- **Confluence** — search/read wiki pages
- **MyWiki** — alternate Confluence instance
- **GitHub** — browse repos, create PRs, review code
- **Mermaid** — generate diagrams

## Updating Rules

When steer-runtime templates are updated:

```bash
cd ~/steer-runtime
git pull
./setup.sh cursor sync ~/my-project
```

## Cursor vs Kiro

| Feature | Kiro CLI | Cursor |
|---------|----------|--------|
| Coding standards | ✅ via agent prompts | ✅ via .mdc rules |
| MCP tools (Jira, Confluence, GitHub) | ✅ | ✅ |
| Multi-agent orchestration | ✅ 40 agents | ❌ single agent |
| Automated SDLC workflow | ✅ 12-step pipeline | ⚠️ manual guidance |
| Programmatic hooks | ✅ guard-writes, git-context | ⚠️ rule instructions |
| Profile switching (dev/ba/qa/ops/pm) | ✅ | ❌ all rules present |
| Persistent memory (knowledge tool) | ✅ | ❌ session only |
| Agent/rule count | ✅ 40 agents, 5 profiles | ✅ 19 rules (glob-activated) |
| Inline code suggestions | ❌ | ✅ Tab completion |
| Multi-file Composer | ❌ | ✅ |
| Codebase indexing | ❌ | ✅ @codebase |

Use both: Kiro for complex orchestrated workflows, Cursor for day-to-day coding with the same standards.
