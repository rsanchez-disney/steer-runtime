## Executive summary

Payment gateway timeouts resumed after a deployment to Payment Service (BAPP0012692) on July 1, 2026. This is the **9th recurrence** of the same root cause: the June 22 connection pool/TLS hotfix is overwritten every time a new deployment executes against this service because the fix was never merged into the main deployment artifact.

**This is no longer a technical incident — it is a process failure.** The same root cause has been identified, documented, and assigned remediation action items that remain unimplemented after 17 days.

## Root cause confirmation

### Immediate cause

A deployment executed on July 1, 2026 against Payment Service (BAPP0012692) replaced the running configuration with the standard deployment artifact, which **does not contain** the June 22 hotfix.

### Underlying cause

The June 22, 2026 hotfix (connection pool sizing + TLS configuration) was applied as a runtime/operational fix but was **never merged into the source branch or deployment artifact** used by the CI/CD pipeline. Every subsequent deployment reverts the service to the broken pre-hotfix state.

### Contributing factors

1. **No deployment guard** — No pre-deployment check validates that the hotfix configuration is present in the artifact being deployed
2. **No deployment freeze** — Despite 8 prior recurrences, no freeze was enforced on Payment Service deployments
3. **Action items not executed** — Remediation items from prior RCAs and sprint retros remain unresolved past their due dates
4. **No automated regression test** — No health check or smoke test validates connection pool/TLS behavior post-deployment

### Expected log pattern (to validate in Splunk)

- `Connection pool exhausted` / `No available connections`
- `TLS handshake timeout` / `SSL handshake failed`

- Spike in connection wait times > configured timeout threshold

## Escalation recommendation

**This incident requires immediate director-level escalation.** Justification:

1. **9 recurrences in 17 days** — The same root cause has now caused 9 outages. This is not a technical gap; it is a systemic failure to execute known remediation.
2. **Action items past due** — The deployment guard was due July 1 (not implemented). The director remediation plan is due today, July 2.
3. **Customer impact is cumulative** — Each recurrence impacts guest bookings and revenue. The aggregate impact over 17 days is significant.
4. **On-call fatigue** — Engineers are repeatedly woken/paged to re-apply the same fix manually.

### Recommended escalation actions

- Director to mandate immediate merge of hotfix into main artifact (today)
- Director to enforce deployment freeze with explicit gate (no deployments without hotfix validation)
- VP-level visibility if hotfix is not merged by EOD July 2
